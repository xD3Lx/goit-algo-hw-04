Ми порівнювали три алгоритми сортування: злиттям, вставками та Timsort (функція sorted) за часом виконання на масивах трьох різних розмірів(500, 5000 та 50000 елементів).
Результати наступні:
- найменш ефективним алгоритмом сортування є сортування вставками, оскільки він має найбільшу тривалість сортування серед трьох алгоритмів;
- середнім за тривалістю виконання є алгоритм сортування злиттям, який показує кращі результати порівняно з сортуванням вставками, але все ж має більший час виконання, ніж Timsort;
- найбільш ефективним в даному випадку є алгоритм сортування Timsort, який має найменший час виконання серед трьох алгоритмів. 
  Це може бути пов'язано з тим, що Timsort є оптимізованим алгоритмом, який використовує різні стратегії для сортування даних, що дозволяє йому працювати швидше в багатьох випадках.
